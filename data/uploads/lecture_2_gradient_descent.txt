Lecture 2: Optimization and Gradient Descent

1. Optimization in Machine Learning
Optimization is the process of finding the parameters (weights W and bias b) that minimize the loss function. The loss function measures how far the model's predictions are from the true labels.

2. Gradient Descent Algorithm
Gradient descent is an iterative optimization algorithm used to minimize a loss function.
The core idea is to start with random weights, and then repeatedly make small steps in the direction of steepest descent (opposite to the gradient of the loss function).
- Update rule: W = W - alpha * dW
  where alpha is the learning rate, and dW is the derivative of the loss function with respect to W (the gradient).

3. The Learning Rate (Alpha)
The learning rate is a hyperparameter that controls how large a step we take at each iteration.
- If the learning rate is too small: The algorithm will take many iterations to converge, making training very slow.
- If the learning rate is too large: The algorithm can overshoot the minimum, bounce back and forth, and fail to converge (diverge).

4. Variants of Gradient Descent
- Batch Gradient Descent: Computes the gradient of the loss function with respect to the parameters for the entire dataset. It is computationally expensive for large datasets.
- Stochastic Gradient Descent (SGD): Computes the gradient and updates parameters for each training sample individually. It is fast but highly noisy.
- Mini-batch Gradient Descent: Computes the gradient on small random subsets of the dataset (mini-batches, e.g., size 32, 64, or 128). It combines the speed of SGD with the stability of Batch Gradient Descent, and is the standard in modern deep learning.
